## 👩‍💻 주요 기능 
- 회원가입시 인증메일 전송
- 네이버 간편가입 기능
- 카카오 지도 API 
- 결제 API
- 결제 완료시 java mail 라이브러리 이용하여 메일 전송 / 카카오톡 메세지 공유 API
- 예약 리스트 및 상세조회

　

## 💡 기능 구현
> ### 회원
- <b>[회원가입 및 로그인]</b> 회원가입시 인증메일 전송 / 네이버 간편가입 / 아이디·비밀번호 찾기
- <b>[마이페이지]</b> 렌트내역 리스트 및 상세조회 / 나의 게시글 작성 및 조회 / 개인정보수정 / 회원탈퇴
- <b>[메인페이지]</b> 추천 차량·리뷰정보 출력 / 차량·회원·예약 통계 / 지점안내 / 사이트소개

 
> ### 예약 및 결제 (✅참여)
- <b>[메인페이지]</b> 예약·반납 일시 및 이용지점 선택
- <b>[차량선택]</b> 차종·지점에 따른 차량선택 기능 / 하단네비게이션바 예약정보 출력 / 리뷰조회 및 댓글작성 / 차량 상세정보 조회
- <b>[예약 및 결제]</b> 예약 정보 및 사용자 정보 출력 / 결제API 통한 결제구현 / 결제완료시 java mail 라이브러리 이용하여 메일 전송 / 카카오톡 메세지 공유 API
- <b>[예약 조회 및 취소]</b> 예약리스트 및 상세조회 / 예약 취소시 예약상태 변경 / 24시간 이전 예약취소 불가 제어

 
> ### 관리자
- <b>[관리자 메인]</b> 예약 현황 조회 / 캘린터이용한 업무일정관리 / 공지사항 관리 / 회원조회 / 블랙리스트 처리
- <b>[차량 관리]</b> 차량리스트 등록·수정·삭제 / 리뷰리스트 및 상세

## :question: 어려웠던 점

> ### 결제 전 예약데이터 저장 방법
- 메인에서부터 고객이 선택한 예약데이터를 결제 직전까지 가져가는 몇 가지 방법에 대해 어떤 것이 더 효율적인지 고민함

> ### 파트 간의 의사소통 문제
- 정해진 기간동안 모든 파트가 유동적으로 돌아가도록 완성하기로 계획했으나, 각자의 파트에만 몰두하느라 각 진행상황이 잘 공유되지 않음.
- 다른 파트의 데이터가 필요한 파트의 경우, 계획의 진도에 차질이 생기는 일이 발생
- 같은 데이터 값을 사용하지만 변동사항이 있을 때 잘 전달이 안되지 않아 일의 효율이 떨어짐

## :exclamation: 해결 방법

> ### 메인부터 결제진행까지 세션에 정보를 저장해서 전달
- 메인 페이지에서 예약 일시, 지점 선택 후 예약 테이블에 먼저 저장을 하면 그 이후에는 계속 update쿼리를 사용해야 함
- 하지만 도중에 예약 페이지를 벗어나거나 예약을 진행하지 않는 경우, 저장된 데이터들을 다시 삭제하는 작업을 해야 하므로 비효율적임
- :arrow_forward: <b>처음부터 결제 진행까지 세션에 저장한 후 결제 완료할 때 세션의 정보들을 예약 테이블에 저장하도록 메서드 작성</b>

> ### 슬랙(Slack)을 이용하여 진행사항 및 변동사항 공유
- 변동사항이 있는 경우 슬랙(Slack)을 이용하여 실시간으로 수정한 파일명 및 수정내용 공유

## :100: 느낀점

> ### 수정과 유지보수를 위해서 주석을 세밀하게 작성
- 자신이 작성한 메서드가 어떤 기능을 하는지, 어떤 파라미터를 사용하는지 등을 자세하게 기재해야 팀원들이 코드를 이해하는 시간을 현저하게 줄일 수 있고 일의 효율을 높일 수 있다는 것을 몸소 느낌

> ### 구체적이고 체계적인 설계가 구현과 유지보수의 대부분을 차지한다
- 설계를 완벽하게 했다고 생각했으나 제대로 고려하지 못한 변수들 때문에 계속해서 수정하는 일이 빈번하게 생겨 계획대로 실행하지 못함
- 다음 프로젝트부터는 급하게 실행하지 않고 설계 단계부터 탄탄하게 해 나아가야겠다 느낌
